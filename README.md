# Storage Lab: open image-compression & cloud-storage datasets

Reproducible benchmarks and open datasets on **image compression (JPEG / WebP / AVIF / JPEG XL), the HEIC-to-JPG size tax, cloud storage $/GB, and phone storage capacity.** Every number here is generated by a script in [`benchmarks/`](benchmarks/) that you can re-run yourself, on your own images, in one command.

Maintained by [Cleanor Labs](https://cleanor.app/research). If a study is useful, please cite it (see [Citation](#citation)) — that is the whole point of publishing the raw data.

## Why this exists

Most "AVIF saves 50%" or "just convert HEIC to JPG" claims online cite each other, not a measurement. We publish the corpus-driven measurement **and the harness that produced it**, so anyone can verify, extend, or disagree with the method rather than the vibe.

The image benchmarks run on the classic **Kodak** and public high-resolution phone-photo corpora, encode with **libvips/sharp** (the same engine behind Cleanor's browser tools) and **libjxl / libheif**, and score perceptual quality with **SSIM + PSNR via ffmpeg** against the lossless master. Nothing is hand-tuned per image.

## The datasets

All CSVs live in [`data/`](data/). Highlights below are pulled straight from those files.

### 1. Image compression reality — JPEG vs WebP vs AVIF
`data/compression-benchmark.csv` · `data/compression-benchmark-summary.csv`

Bytes needed to reach the **same perceptual quality (SSIM)** across a full quality ladder, averaged over the corpus:

| Target quality | JPEG | WebP | AVIF | AVIF vs JPEG |
|---|---|---|---|---|
| SSIM 0.95 ("good") | 1,078 KB | 939 KB | **641 KB** | **−41%** |
| SSIM 0.98 ("high") | 1,419 KB | 1,777 KB | **1,008 KB** | **−29%** |
| SSIM 0.99 ("visually lossless") | 1,545 KB | 2,427 KB | **1,211 KB** | **−22%** |

Takeaway: AVIF wins at every quality tier, but WebP is *worse than JPEG* above ~SSIM 0.98 — the "WebP is always smaller" folklore is false at high quality.

### 2. State of next-gen formats — + JPEG XL
`data/nextgen-formats-benchmark.csv`

Adds **JPEG XL** to the matrix, including its killer feature: **bit-exact lossless recompression of an existing JPEG** (`jxl_lossless_bytes` vs `jpeg_q90_bytes`), which shrinks legacy JPEGs ~20% with zero quality loss and full reversibility.

### 3. The HEIC Tax — what "heic to jpg" really costs
`data/heic-tax-benchmark.csv` · `data/heic-tax-summary.csv`

246,000 people/month Google "heic to jpg". Converting an iPhone HEIC to JPG at the quality online converters use inflates the file with **no visible quality gain**:

| iPhone HEIC quality | Mean SSIM | JPG size tax (median) | PNG size tax (median) |
|---|---|---|---|
| High (SSIM 0.99) | 0.987 | **1.75× larger** | 3.87× larger |
| Medium (SSIM 0.98) | 0.978 | 2.51× larger | 5.50× larger |

Converting the same photos to PNG can bloat them **13×** for zero perceptual benefit.

### 4. Cloud storage price index
`data/cloud-price-index.csv`

$/GB/month and the 10-year "storage rent" across Apple iCloud+, Google One, Microsoft 365, Dropbox, Amazon, Proton, Box, pCloud (lifetime) and Backblaze. Includes the **small-tier penalty**: iCloud's 50 GB tier costs **4× more per GB** than its 2 TB tier.

### 5. Supporting reference datasets
- `data/streaming-data-usage.csv` — MB/hour for Netflix, YouTube, Disney+, TikTok, etc. (official + measured)
- `data/photo-storage-capacity.csv` — how many HEIC / JPEG / 48 MP / ProRAW photos fit in each iPhone tier (advertised vs usable GB)
- `data/passport-photo-specs.csv` — passport/visa photo specs (mm, head size, background, digital limits) by country
- `data/storage-trends-*.csv` — search demand for storage-cleanup queries by country and month

## Reproduce it yourself

```bash
git clone https://github.com/cleanor-app/cleanor-storage-lab.git
cd cleanor-storage-lab
npm install                      # installs sharp

# Drop your own .png/.tif/.jpg masters here:
mkdir -p corpus && cp /path/to/your/images/*.png corpus/

# Run any study (writes to data/):
node benchmarks/compression-benchmark.mjs
node benchmarks/nextgen-formats-benchmark.mjs
node benchmarks/heic-tax-benchmark.mjs
node benchmarks/cloud-price-index.mjs     # no corpus needed
```

**System dependencies** (macOS `brew`, Linux `apt`):
- `ffmpeg` — SSIM/PSNR scoring (all image studies)
- `libheif` (`heif-enc`, `heif-convert`) — the HEIC study
- `jpeg-xl` (`cjxl`, `djxl`) — the next-gen study

`sharp` (libvips) is installed by `npm install`. See [`docs/methodology.md`](docs/methodology.md) for the exact method, corpus, and caveats.

## Column reference (image benchmarks)

`image, width, height, pixels, format, quality, bytes, bpp, ssim, psnr`
where `ssim`/`psnr` are measured against the lossless master and `bpp` is bits-per-pixel. Higher SSIM/PSNR = closer to the original.
